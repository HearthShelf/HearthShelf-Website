Installation

HearthShelf ships as a single Docker image. It runs as a warm web front-end over your existing Audiobookshelf server — point it at your library, give it a config volume, and you're listening in under two minutes.

lightbulbTip
Already running Audiobookshelf? You don't need to migrate anything. HearthShelf reads the same library and database — it's purely a nicer way in.

Prerequisites

  • Docker Engine 24+ and the Compose plugin
  • A running Audiobookshelf server (local or remote)
  • A folder of audiobooks mounted on the host

Docker Compose Recommended

Create a docker-compose.yml and adjust the audiobooks path to match your host:

docker-compose.yml content_copy 
services:
  hearthshelf:
    image: ghcr.io/hearthshelf/hearthshelf:latest
    container_name: hearthshelf
    ports:
      - "8200:80"
    volumes:
      - ./config:/config
      - ./metadata:/metadata
      - /path/to/audiobooks:/audiobooks
    environment:
      - ABS_URL=http://audiobookshelf:13378
      - TZ=America/New_York
    restart: unless-stopped

Then bring the stack up:

$ docker compose up -d

First run

Open http://localhost:8200 in your browser. On first launch HearthShelf walks you through connecting your Audiobookshelf server and signing in — your library, progress, and bookmarks sync straight across.

Reverse proxy

warningWarning
HearthShelf streams audio over long-lived connections. If you put it behind Nginx or Caddy, raise proxy_read_timeout and disable buffering, or chapter seeks may stall.

Updating

Pull the latest image and recreate the container. Your config volume is untouched.

$ docker compose pull
$ docker compose up -d
